## 备忘

优秀的 Git Commit 信息的 7 条规则：

1. 使用空行将主题与正文分开；
2. 将主题行限制为 50 个字符；
3. 主题行大写；
4. 不要以句号结尾主题行；
5. 在主题行中使用祈使语气；
6. 正文在 72 个字符处换行；
7. 使用正文来解释What、Why 和 How。
